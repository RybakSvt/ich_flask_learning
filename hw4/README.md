# Домашнее задание: Работа с базой данных SQLite using SQLAlchemy ORM

## Описание задания
Реализация CRUD операций (Create, Read, Update, Delete) и агрегаций для базы данных интернет-магазина с использованием SQLAlchemy ORM.

## Функциональность
- Создание моделей данных для продуктов и категорий
- Наполнение базы данных тестовыми данными
- Выполнение запросов на чтение, обновление и агрегацию данных
- Работа со связями между таблицами

## Модели данных

### Category (Категории)
- `id`: числовой идентификатор (первичный ключ)
- `name`: название категории (строка, 100 символов)
- `description`: описание категории (строка, 255 символов)
- `products`: связь с таблицей Product (один ко многим)

### Product (Продукты)
- `id`: числовой идентификатор (первичный ключ)
- `name`: название продукта (строка, 100 символов)
- `price`: цена продукта (число с фиксированной точностью)
- `in_stock`: наличие на складе (логическое значение)
- `category_id`: внешний ключ на таблицу Category
- `category`: связь с таблицей Category (многие к одному)

## Задачи

### Задача 1: Создание структуры базы данных
- Создание моделей и таблиц в SQLite базе данных
- Настройка связей между таблицами

### Задача 2: Чтение данных
- Извлечение всех категорий с связанными продуктами
- Вывод информации о продуктах для каждой категории

### Задача 3: Обновление данных
- Поиск продукта по названию
- Обновление цены продукта

### Задача 4: Агрегация и группировка
- Подсчет количества продуктов в каждой категории
- Использование агрегирующих функций

### Задача 5: Группировка с фильтрацией
- Фильтрация категорий по количеству продуктов
- Вывод только тех категорий, где более одного продукта

## Связь между таблицами
Таблица `Product` содержит внешний ключ `category_id`, который связывает её с таблицей `Category`.  
Одна категория может включать несколько продуктов (отношение **один ко многим**).
